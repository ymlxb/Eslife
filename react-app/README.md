# EcoLife - 绿色生活助手平台

> 🔗 **GitHub 仓库**: [https://github.com/ymlxb/chuanzhi-cup](https://github.com/ymlxb/chuanzhi-cup)  
> (注：本项目代码位于该仓库的 `main-react` 分支下的 `my-vue-app/react-app` 目录中)

## 📖 项目介绍

**EcoLife** 是一个基于 React 19 和 TypeScript 开发的综合性环保生活服务平台。项目旨在通过数字化手段倡导低碳生活方式，集成了 **AI 环保助手**、**个人碳足迹计算**、**二手闲置交易**以及**环保社区**等核心功能。用户不仅可以量化自己的环保贡献，还能通过闲置物品流转和社区互动，实际参与到绿色循环经济中。

本项目采用现代化的前端技术栈构建，注重用户体验与代码的可维护性，是一个功能完整、交互丰富的单页应用 (SPA)。

## 🛠️ 技术栈

- **核心框架**: [React 19](https://react.dev/) - 采用最新的 React 特性构建用户界面。
- **构建工具**: [Vite](https://vitejs.dev/) - 极速的开发服务器与打包工具。
- **开发语言**: [TypeScript](https://www.typescriptlang.org/) - 强类型约束，提升代码质量与开发效率。
- **UI 组件库**: [Ant Design](https://ant.design/) - 企业级 UI 设计语言和 React 组件库。
- **状态管理**: [Redux Toolkit](https://redux-toolkit.js.org/) - 高效的全局状态管理方案。
- **路由管理**: [React Router DOM](https://reactrouter.com/) - 处理应用内的页面跳转与路由守卫。
- **数据可视化**: [ECharts](https://echarts.apache.org/) - 用于展示碳足迹统计图表。
- **网络请求**: [Axios](https://axios-http.com/) - 封装统一的请求拦截与响应处理。
- **样式预处理**: [Less](https://lesscss.org/) - 编写模块化、可复用的 CSS。
- **Markdown 渲染**: marked + DOMPurify - 安全地渲染 AI 助手的富文本回复。

## ✨ 核心功能

### 1. 🤖 AI 环保助手

集成智能对话功能，为用户提供实时的环保知识问答。

- 支持流式对话体验。
- 解答垃圾分类、节能减排技巧等问题。
- Markdown 格式渲染，支持富文本展示。

### 2. 👣 碳足迹计算器

可视化的个人碳排放追踪工具。

- **多维度计算**: 支持电力、燃气、交通（私家车/公交/电车）等维度的碳排放计算。
- **数据可视化**: 使用 ECharts 绘制碳排放趋势图与占比图，直观展示环保成效。
- **历史记录**: 本地存储计算历史，方便用户对比分析。

### 3. ♻️ 二手闲置交易

构建绿色循环经济的交易市场。

- **商品分类**: 覆盖数码、图书、服饰、家居等多个品类。
- **商品发布**: 用户可便捷发布闲置物品，支持上传图片与详细描述。
- **商品检索**: 支持按名称、标签搜索商品。
- **详情展示**: 包含商品大图预览、详细信息及卖家联系方式。

### 4. 👥 环保社区与资讯

- **社区互动**: 用户分享环保心得，浏览他人动态。
- **环保指南**: 提供各类环保生活小贴士与官方指南。
- **品牌专区**: 展示环保合作品牌与绿色产品。

### 5. 👤 个人中心

- **账户管理**: 头像上传、密码修改、收货地址管理。
- **订单管理**: 查看买入与卖出的订单状态。
- **发布管理**: 管理自己发布的闲置商品与社区帖子。

└── main.tsx        # 入口文件

## 🚀 快速开始

### 环境要求

- Node.js >= 16
- npm 或 yarn

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

启动后访问 `http://localhost:5173` (或控制台显示的局域网地址)。

### 构建生产版本

```bash
npm run build
```

## 💡 项目亮点 (面试/简历可用)

1.  **TypeScript 全面应用**: 项目全量使用 TypeScript 开发，定义了完善的接口类型（如 `ApiResponse`, `EnergyData` 等），显著提升了代码的健壮性与可维护性。
2.  **组件化与模块化**: 封装了通用的网络请求模块（Axios Interceptors）、懒加载图片组件 (`LazyImage`) 以及统一的布局组件，实现了高内聚低耦合的代码结构。
3.  **数据可视化实践**: 结合 ECharts 实现动态数据渲染，将抽象的碳排放数据转化为直观的图表，提升了用户对环保数据的感知度。
4.  **性能优化**:
    - 使用 Vite 进行快速构建与热更新。
    - 图片资源懒加载优化首屏体验。
    - 路由懒加载（可配置）减少首屏包体积。
5.  **AI 场景落地**: 将 AI 技术与垂直领域（环保）结合，通过流式对话与 Markdown 渲染，提供了良好的交互体验。
